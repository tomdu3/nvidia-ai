### 🔑 **Core Philosophy First**

> **"Learn the _why_ before the _how_.** Focus on intuition, math basics (linear algebra, stats, calculus), and problem formulation _before_ diving into frameworks. AI tools are useless without understanding bias/variance, overfitting, or when to use which algorithm."

---

### 📚 **One-Line References (Bookmark These)**

_(Prioritize depth over quantity—master 1-2 before moving on)_

1. **[Andrew Ng's ML Course (Stanford/Coursera)](https://www.coursera.org/learn/machine-learning)** → _The gold standard for intuition-first ML (uses Octave/MATLAB; math-light but profound)._
2. **[Hands-On Machine Learning with Scikit-Learn, Keras & TensorFlow (Aurélien Géron)](https://www.oreilly.com/library/view/hands-on-machine-learning/9781492032632/)** → _Free [legally via GitHub](https://github.com/ageron/handson-ml2); best practical guide (code-first, minimal math)._
3. **[Deep Learning Book (Goodfellow, Bengio, Courville)](https://www.deeplearningbook.org/)** → _Free authoritative DL bible (read Ch 1-6 for ML foundations; math-heavy but essential)._
4. **[StatQuest with Josh Starmer (YouTube)](https://www.youtube.com/@statquest)** → _Non-negotiable for stats intuition (p-values, distributions, PCA—watch his "Machine Learning Basics" playlist)._

> 💡 **Tip**: Skip ML "cheat sheets" early on. Build mental models first—Ng’s course + Géron’s book cover 80% of real-world needs.

### 🛠️ **Free AI Tools for Learning (Use _After_ Basics)**

_These reinforce concepts—don’t replace learning theory._

1. **Google Colab** → _Free GPU/Jupyter notebooks (run Géron’s book code here; [tutorial](https://colab.research.google.com/notebooks/intro.ipynb))._
2. **Kaggle Learn** → _Free micro-courses (Pandas, ML, DL); compete in beginner [Titanic](https://www.kaggle.com/titanic) to apply skills._
3. **Hugging Face Spaces** → *Experiment with open-source models (e.g., [Spaces](https://huggingface.co/spaces))—*see* how transformers work after learning basics.*
4. **Scikit-learn Documentation** → _The [user guide](https://scikit-learn.org/stable/user_guide.html) is a textbook—work through tutorials._
5. **TensorFlow Playground** → _Visualize neural nets in-browser ([link](https://playground.tensorflow.org/))—build intuition before coding._

> ❌ **Avoid premature tool-hopping**: Don’t jump to LLMs/APIs (like OpenAI) until you grasp gradient descent, loss functions, and evaluation metrics. Tools amplify skill—they don’t create it.

### 🚫 **What to Skip (For Now)**

- Advanced LLMs/Prompt Engineering (learn ML basics first—this is applying ML, not learning it).
- Math-heavy proofs (save for after you grasp concepts intuitively).
- "AI tool" tutorials that hide the ML behind APIs (e.g., "Use this API to detect cats").

This path builds **durable skills**—not just clicking buttons. ML mastery comes from wrestling with concepts, not collecting certificates. Start small, stay consistent, and _build something_ every week (even if it’s just reproducing a tutorial). You’ve got this! 💪
